Writing Documents in XML

This is a collection of materials related to creating documents using XML. We write our documents such as articles, tutorials, assignments and books in Docbook, an XML vocabulary for technical documents. We convert these to HTML, PDF and LaTeX.

There are many benefits to using Docbook/XML in preference to other markups (e.g. LaTeX). These include

  • separating content and meaning from appearance,
  • targetting HTML and Web browsers which are increasingly richer than PDF,
  • synchronizing content across documents and formats and being able to reuse content,
  • being able to combine segments of different documents easily into another document,
  • being able to programmatically verify and modify structured documents,
  • using XSL templates to customize how the document is rendered
  • using XML tools which we increasly use in other settings, e.g. Web services, scraping Web pages, reading structured data.

We are not trying to convinvce people to abandon LaTeX for a new toolchain. We are merely attempting to make people aware of a rich alternative. This is especially applicable for people just embarking on a career and finding a good working setup and environment.

    • Simple two-stage dynamic documents

    We can easily make XML documents dynamic in the sense of evaluating code contained in the document and inserting the results into the projected view of the document. This is like Sweave, but with a very extensible document structure and processing engine. (In addition to dynamic content, we can make these documents richly interactive.)

    There are two approaches to dynamic documents with XML and XSL.

    • One is to process the document in one step that converts the XML to the target format (e.g. HTML or FO or LaTeX), and evaluates the R code (or other languages) via XSL rules and R functions as extension XSL functions
    • The second is to have two stages: process the XML document in R and evaluate the R code and insert the results, write the results to new document and then process that with the regular XSL conversion tools.
